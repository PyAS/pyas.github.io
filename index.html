>>> i just learned writing Math in IPy NB.

— Written on March 2, 2015 | Also at Pyas.Me

This is my experience while learning to write mathematical expressions in IPython NoteBook in such a way that when I convert this into html, it should still work. I found the necessary source (mentioned below) and tried it. But I had to go through a rigorous strain in getting my html file render the math expressions as intended.

Lessons Learned !

I learned pretty serious lessons on my way of coming up with this webpage. I first went on typing all that I saw on this page line by line. On my local machine, every output was displayed like absolutely what it was suposed to be. But when I converted this .ipynb file into html using ipython nbconvert command and opened the newly created html file using a browser, I was surprised to find that none of the outputs worked. There were no mathematical equations at all. All I could see was the raw TEX commands. Before doing this whole check, I got excited and deployed the html page on my website running on Amazon AWS instance. It was disappointing!

Initially, I tought that I should have not installed something that would enable a perfect html output. I was right about that. I googled and found that I was supposed to have MathJax.JS in my machine and the html should have acess to this JS file while loading in order to render math expresions correctly. Without knowing what went wrong, I spent some time upgrading my ipython locally and on my server. I tried to install on my server stuff like pyzmq and nose without any least idea what they were meant to be. Honestly, I am still not sure if installing and testing MathJax on my machine and in my server helped me or not. But a simple code replacement in my html file did surely make the whole difference.

When I converted IPython Notebok file (.ipynb) into html, it is somehow taking some default JavaScript path which doesn't seem to be working. I then had to chat on Gitter.Im's IPython Help Chat Room and take inputs from peers to do this whole work. Especially, Eliott's following comment showed me what went wrong.

The MathJax script URL is 404'ing on your website In nbviewer, it uses https://cdn.mathjax.org/mathjax/latest/MathJax.js?config=TeX-AMS_HTML, but yours seems to be some specific mirror https://c328740.ssl.cf1.rackcdn.com/mathjax/latest/MathJax.js?config=TeX-AMS_HTML

And the Documentation on MathJax site gave me a hint on this which I couldn't instantly understand at the first glance.

You load MathJax into a web page by including its main JavaScript file into the page. That is done via a <script> tag that links to the MathJax.js file. To do that, place the following line in the <head> section of your document:

<script type="text/javascript" src="path-to-MathJax/MathJax.js"></script>

where path-to-MathJax is replaced by the URL of the copy of MathJax that you are loading. For example, if you are using the MathJax distributed network service, the tag might be

<script type="text/javascript" src="http://cdn.mathjax.org/mathjax/latest/MathJax.js"></script>

If you have installed MathJax yourself, path-to-MathJax will be the location of MathJax on your server, or (if you are using MathJax locally rather than through a server) the location of that directory on your hard disk. For example, if the MathJax directory is at the top level of your web server’s directory hierarchy, you might use

<script type="text/javascript" src="/MathJax/MathJax.js"></script>

to load MathJax.

That's it! I thought I should look for this wrong place in my html and replace it with a correct link and I would be done with it.

Not yet!

I ended up with another issue! This time, the last couple cells in this page would display the quations perfectly, but none of the others. I tried my best to understand what went wrong again. I finally gave up and relied on StackOverflow this time. Thanks to Jakob who explained that ipython nbconvert uses pandoc which requires each equation to be written within the delimiters $$ ... $$. This point is in fact mentioned in the same source website, but I failed to notice it due to my ignorance with this subject. I am glad I am making it up in this new arena and I can see that it is going to be a hell of a ride!

Finaly, everything worked.

So the take away is:

  1. Embedding the right MathJax.js source link is very essential for this to work.
  2. The $$ ... $$ delimiters for each equation are very necessary if an .ipynb file is nbconverted-converted into .html via pandoc.

Finally, everything works just fine!!!

  • The only exception being the $$ ... $$ delimiters covering each equation. Thanks to the creators!

    'o'which inserts a line and puts you in --INSERT-- mode enabling you to write the content.
    'O'which doest the same thing but inserting a line before your current position.

And the lesson ends with a line that says,

I bet you feel like O_ _ _o

And I truly felt that.

And the other truly interactive lessons include:

1. VIM - 3 Incarnations:

    VIM's Normal mode: Lets you use all the VIM commands/shortcuts.
    VIM's --INSERT-- mode: Lets you write and edit content.
    VIM's --VISUAL-- mode: Lets you select text and manipulate.

2. Write (edit), Conquer (save) & Go (quit):

  • Type :w to save the work.
  • Type :q to quit VIM after saving.
  • Type :q! to quit without saving.
  • Type :wq to save and quit.
  • Add ! to force these commands.

3. Plucking out the arrow keys on your keyboard:

  • Moving your cursor in normal mode:
    • h for moving left
    • l for moving right
    • j for moving down
    • k for moving up

4. Letters speak louder than Words:

  • Moving within your words in normal mode:
    • w for the first letter of the next Word
    • b for the beginning of the word
    • e for the end of the word

5. A Shocking Movement: w/ Numbers & Symbols

  • Numbers in VIM's normal mode have additional meanings:
    • Typing 9w is pressing w nine times.
    • 6l is pressing l six times.
    • 5iAnand inserts my name Anand five times.
    • % takes you to parenthesis.
    • 0 and $ take you to the beginning and the end of a line, respectively.
    • gg takes you to the beginning of the file.
    • G takes you to the end of the file.
    • And 7G takes you to the line-7 in the file.
